# Ciao, GitHub Actions

_Crea ed esegui un workflow di GitHub Actions._

## Benvenuto

L'automazione è fondamentale per attività ripetitive come test, scansioni, revisioni e deployment. [GitHub Actions](https://docs.github.com/actions) ti aiuta a gestirle direttamente nel repository e a rendere più efficiente il tuo workflow.

- **A chi è rivolto**: Studenti, sviluppatori, ingegneri DevOps e ingegneri della sicurezza
- **Cosa imparerai**: Come creare workflow di GitHub Actions, eseguirli e usarli per automatizzare le attività.
- **Cosa costruirai**: Un workflow di GitHub Actions che aggiunge un commento a una pull request quando viene aperta.
- **Prerequisiti**: [Introduzione a GitHub](https://github.com/skills/introduction-to-github)
- **Quanto tempo**: Questo esercizio può essere completato in meno di 30 minuti.

In questo esercizio:

1. Creerai il file del workflow
1. Aggiungerai un job
1. Aggiungerai uno step di esecuzione (run step)
1. Controllerai il workflow in esecuzione
1. Eseguirai il merge della tua pull request

### Come iniziare questo esercizio

1.  Fai clic con il tasto destro su **[Step 0](../../actions/workflows/0-start-exercise.yml)** e aprilo in una nuova scheda.
2.  Fai clic su **Run workflow**.
3.  Seleziona il branch `main` e fai clic sul pulsante verde **Run workflow**.
4.  Attendi circa 20 secondi, poi aggiorna questa pagina.
